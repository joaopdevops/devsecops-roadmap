# Lab: 1.1 - Redes no Mundo Real

**Data:** 15/04/2026
**Mes:** 01 - Redes Fundamentos
**Fase:** Fase 1 - CCNA
**Duracao:** ~1h30

---

## Objetivo

Montar uma LAN do zero no Packet Tracer com 2 PCs e 1 Switch,
configurar IPs manualmente e testar comunicacao com ping.

---

## Comandos Utilizados

| Comando | O que faz |
|---------|-----------|
| `ping 192.168.1.2` | Testa comunicacao entre dois dispositivos via ICMP |
| `ping -c 4 192.168.1.2` | Ping no Linux — envia 4 pacotes e para |
| `ip addr show` | Mostra todas as interfaces de rede e seus IPs |
| `ip route show` | Mostra a tabela de roteamento do sistema |
| `sudo ip addr add 192.168.1.1/24 dev wlp0s20f3` | Adiciona IP manualmente a uma interface |

---

## O que montei no Packet Tracer

```
PC0 (192.168.1.1) ----[cabo]---- Switch0 ----[cabo]---- PC1 (192.168.1.2)
```

**Dispositivos:**
- 2x PC-PT
- 1x Switch 2960-24TT
- Cabo: Copper Straight-Through

**IPs configurados:**
- PC0: 192.168.1.1 / 255.255.255.0
- PC1: 192.168.1.2 / 255.255.255.0

**Resultado do ping:**
```
Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)
```

## O que aprendi

**LAN (Local Area Network):** rede local onde dispositivos se comunicam
diretamente sem precisar de roteador.

**Switch:** funciona como carteiro do predio. Recebe o pacote de um PC
e entrega para o outro. Nao pensa, so entrega. Opera na Camada 2 do modelo OSI.

**Roteador:** conecta redes DIFERENTES. E a agencia dos Correios que
liga um predio ao outro.

**IP Address:** endereco unico de cada dispositivo na rede.
Exemplo: 192.168.1.1 = Apto 1 do Predio 192.168.1

**Mascara de rede (255.255.255.0 = /24):** define qual e o "prédio".
Dispositivos com o mesmo predio se comunicam direto.
Se o predio for diferente, precisa de roteador.

**ICMP:** protocolo usado pelo ping. Nao transfere dados — so testa
se o caminho esta aberto.
- Echo Request: "ei, voce esta ai?"
- Echo Reply: "estou sim!"
- 0% packet loss = comunicacao perfeita

**Minhas interfaces Linux:**
- lo (127.0.0.1): loopback — interface virtual interna
- wlp0s20f3 (192.168.0.19): meu Wi-Fi — nome moderno do wlan0
- docker0 (172.17.0.1): rede virtual criada pelo Docker

## Erros que cometi

| Erro | Causa | Solucao |
|------|-------|---------|
| `Cannot find device "eth0"` | Ubuntu moderno nao usa mais eth0 | Usar o nome real da interface: wlp0s20f3 |
| ping 192.168.1.2 falhou no Linux | IP so existia no Packet Tracer, nao na rede real | Normal — sao ambientes diferentes |
